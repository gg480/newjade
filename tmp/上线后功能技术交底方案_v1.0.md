## 1. 项目运行基线

- 技术栈：`Next.js` + `TypeScript` + `Prisma` + `SQLite` + `Tailwind` + `shadcn/ui`
- 运行形态：前后端一体化，API 通过 `src/app/api` 提供
- 数据源：SQLite（线上持久化目录映射）
- 当前结论：核心业务链路可用，已满足上线运行

## 2. 已运行功能范围（业务口径）

### 2.1 商品与库存

- 商品创建、编辑、删除（含软删除语义）
- 多维筛选、排序、详情侧滑、图片查看
- 批量操作（导出、调价、批量选择）
- 批次关联入库、成本分摊展示

### 2.2 销售与退货

- 单件销售、销售编辑、销售查询
- 销售退货流程（退款金额、退货原因、日期）
- 套装销售
- 销售报表与 CSV 导出

### 2.3 客户与供应商

- 客户 CRUD、搜索、消费统计展示
- 供应商 CRUD、关联检查（有关联数据时删除保护）

### 2.4 看板与统计

- 汇总看板、趋势分析、分布分析
- 聚合接口已运行（含 summary/aggregate 等）
- 多图表可视化与移动端适配

### 2.5 导入导出与备份

- 库存 CSV 导入、销售 CSV 导入（已实现）
- 多类数据导出（库存/销售/日志等）
- 数据库备份下载与恢复上传（含基础校验）

### 2.6 日志与设置

- 操作日志记录与查询
- 系统配置管理（店铺名等关键配置）
- 设置页字典管理（材质/器型/标签等）

## 3. 计划分期对照现状（用于产品规划）

### 阶段 1（角色/权限）

- 现状：登录与会话能力已具备
- 差距：角色模型与权限点（字段级、按钮级）未形成完整 RBAC
- 影响：敏感字段（成本/毛利）尚未实现角色隔离

### 阶段 2（销售管理增强）

- 现状：销售编辑、退货、日志已可用
- 差距：调价策略、按角色调价范围控制未落地

### 阶段 3（图片管理优化）

- 现状：上传、展示、封面等能力可用
- 差距：批量替换/批量删除/失败重试/上传队列待完善

### 阶段 4（统计口径与导入收敛）

- 现状：看板接口丰富，聚合能力已上线
- 差距：真实生产 CSV 全量回归需继续强化（按任务清单执行）

### 阶段 5（备份恢复运维化）

- 现状：备份与恢复已能运行
- 差距：恢复演练模式、版本校验、恢复审计记录待补

## 5. 二开与 AI 修改执行规范

- 只允许在明确任务范围内改动，禁止顺手跨模块重构
- 禁止修改既有 API 响应结构（可增字段，不可破坏兼容）
- 涉及导入、销售、备份时必须先执行回归清单
- Schema 变更必须单独评审并记录迁移影响
- 每次改动需同步更新变更记录文档（`CHANGELOG.md` / 任务队列）
