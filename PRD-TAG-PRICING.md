## 一、背景

### 1.1 当前定价方式

现有定价引擎仅基于成本价 × 固定加价倍率，完全不考虑品质差异。两条同材质、同器型、同克重的翡翠，因品质不同价格可差 5~20 倍。

### 1.2 目标（ERP 范围）

- 标签增加 `qualityScore` 字段（0~10），作为品质描述的基础数据
- 入库/出库时展示品质评分 + 相似品成交参考，辅助定价决策
- **不在此项目实现**：学习模型、本地偏好分析、自动调参 → 另立项

## 三、品质评分公式

```
品质总分 = Σ(该分组最高标签分 × 分组权重)
品质分数 = 品质总分 / 理论最高分 × 100
品质倍率 = 下限 + (品质分数 / 100) × (上限 - 下限)
```

同一分组多标签时取最高分（防刷分）。`isGlobal=true` 的通用标签不参与计算。

### 3.1 各材质权重

#### 翡翠/玉

| 分组 | 权重 |
|------|------|
| 种水 | 1.0 |
| 颜色-绿色系 | 0.9 |
| 颜色-其他色系 | 0.8 |
| 透明度（水头） | 0.7 |
| 质地（结构） | 0.6 |
| 净度与瑕疵类型 | 0.6 |
| 颜色分布形态 | 0.5 |
| 种的老嫩 | 0.5 |
| 荧光与光学效应 | 0.4 |
| 工艺类型 | 0.3 |
| 雕刻题材 | 0.2 |
| 器型风格 | 0.1 |

#### 贵金属

克重 × 金价为基础成本，工艺为溢价因子：

```
建议售价 = 克重 × 金价 × (1 + 工艺溢价率) × 成色系数
工艺溢价率 = Σ(工艺标签分 × 工艺权重) / (10 × 工艺权重) × 工艺溢价上限
```

#### 水晶

| 分组 | 权重 |
|------|------|
| 晶底 | 1.0 |
| 色泽 | 0.7 |
| 特殊光学 | 0.5 |

#### 文玩

|------|------|

## 四、API

### 4.1 `POST /api/pricing/quality`

```typescript
// 请求 { tagIds: number[], materialId: number, costPrice?: number, spec?: {...} }
// 响应 { qualityScore, qualityMultiplier, breakdown[], suggestedPrice }
```

### 4.2 `GET /api/pricing/similar?materialId=&typeId=&tagIds=&weight=`

返回同材质 + 同器型的已售货品，按标签 Jaccard 相似度排序取 Top 5，输出成交价范围。

### 4.3 `GET /api/pricing/weights?category=玉`

返回该材质大类的分组权重。

## 六、种子数据

为全部 57 个非通用标签预设 qualityScore，原则：

| 规则 | 示例 |
|------|------|
| 同组相对排序 | 玻璃种=10 > 冰种=8 > 糯冰=6 > 糯=4 > 豆=2 |
| 负面标低分 | 有绺裂=1, 棉多=3 |
| 中性标中等 | 机雕=3, 山水=5 |
| 通用标签不设分 | 热卖/新品/促销 → null |
